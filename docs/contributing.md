# Contributing

Mikro.js is open source and contributions are welcome. Whether it's a bug fix, a new driver, a board definition, or improving the docs, we appreciate the help.

## Ways to contribute

- **Report bugs**: open an [issue](https://github.com/mikrojs/mikrojs/issues) with steps to reproduce
- **Fix bugs**: pick up an existing issue and submit a pull request
- **Write drivers**: create a [driver package](/develop/creating-drivers) for a sensor, display, or peripheral
- **Add board support**: define a [board package](/develop/creating-boards) for a development board
- **Improve docs**: fix typos, clarify guides, or add examples

## Getting started

See the [development guide](/develop/) for instructions on cloning the repo, building from source, and running tests. For the full technical contributing guide (prerequisites, build commands, testing, code style), see [`CONTRIBUTING.md`](https://github.com/mikrojs/mikrojs/blob/main/CONTRIBUTING.md).

## Pull requests

- Keep pull requests focused on a single change
- Include a clear description of what the change does and why
- Make sure `pnpm precommit` passes before submitting
- Add tests where it makes sense

## Code of conduct

We are committed to providing a friendly, inclusive, and welcoming experience for everyone. See our [code of conduct](https://github.com/mikrojs/mikrojs/blob/main/CODE_OF_CONDUCT.md) for details.
